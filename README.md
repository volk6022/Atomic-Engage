# Atomic Engage

Автоматизация Telegram через **живой юзер-аккаунт** (не бот): сообщения, чтение каналов,
follow-up-цепочки, прогрев и повторяющиеся действия по правилам. Часть экосистемы **Atomic**.

> Публичная прод-версия. Полный функционал распространяется по отдельной договорённости.

## Возможности

- **Продажи и лидогенерация** — первое сообщение, follow-up по расписанию, фиксация ответов.
- **Прогрев аудитории** — вступление в каналы, реакции, последовательные касания.
- **Мониторинг** — отслеживание постов/упоминаний по ключевым словам, уведомления в чат/webhook.
- **Безопасность** — гейты по бану/флуду/гео/ASN, дневные бюджеты, гуманизация действий,
  сжатие поведенческого времени (`TIME_SCALE`).

## Архитектура

FastAPI-шлюз · arq-воркеры + watchers · PostgreSQL (asyncpg/alembic) · Redis ·
kurigram (pyrogram-совместимый). Интегрируется с n8n / MCP и другими инструментами.

Прод-модель — **изоляция на клиента**: у каждого клиента свои Postgres/Redis/API_KEY/
прокси-пул (бан или флуд одного не задевает других).

## Быстрый старт

```bash
cd fleet_manager
cp .env.example .env         # заполнить DATABASE_URL, REDIS_URL, API_KEY
docker compose build
docker compose up -d
curl http://localhost:8000/health
```
