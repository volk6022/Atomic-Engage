# Как вносить изменения

Прод-репозиторий. Все изменения проходят через ревью владельца.

## Поток

1. **Issue.** Опиши задачу/баг отдельным issue до начала работы.
2. **Branch.** Ветка от `main`: `feat/<коротко>`, `fix/<коротко>`, `docs/<коротко>`, `chore/<коротко>`.
3. **Commits.** Атомарные, в настоящем времени: `fix: …`, `feat: …`, `docs: …`.
4. **Pull Request.** PR в `main` со ссылкой на issue. CI зелёный.
5. **Review.** Обязательный аппрув владельца. Прямой push в `main` запрещён.

## Правила

- **Секреты и сессии** — только локально, в `.gitignore`: `.env`, `*.session`, `tdata/`,
  прокси-листы, ключи. Перед PR проверь `git diff` на утечки.
- **Прод ≠ дев.** Полный функционал — в приватном upstream `kurigram_for_n8n`.
- **Безопасность аккаунтов** — не ослабляй гейты воркера (`base_task`) без явного решения.
- **Тесты** — гоняй `pytest -m "not accelerated"` перед PR (нужны PG + Redis, см. `CLAUDE.md`).

## Локальный старт

```bash
cd fleet_manager
cp .env.example .env
docker compose up -d postgres redis
```
